Questhub API

Questhub API is open. In fact, the entire website is built using the same API.

Technical details

API routes which require authentication require API tokens; you can get one from your settings. Here's an example on how to use tokens:

http://questhub.io/api/current_user?api_user=your_login&api_token=0123456789abcdef0123456789abcdef

Non-modifying routes, such as ones for obtaining quests or activity feeds, usually don't require authentication.

All API routes are described in API.md file in our github repo.

API is not entirely stable and changes often, but usually in a backward-compatible way. If you want to know about upcoming changes, you should consider following the Meta realm.

Rules

Things to do with API:

  • command-line clients;
  • alternative frontends;
  • exporting data anywhere you like;
  • alternative input methods: email-to-questhub, twitter-to-questhub;
  • importing from relevant sources with high signal/noise ratio (be careful about this one and read below).

Things not to do with API:

  • don't create quests automatically for insignificant achievements
    (for example, don't claim each commit as a quest);
  • don't import from large-volume sources where you don't have the control over quality
    (for example, if your open source project is so popular that people create several issues per day with minor feature requests, maybe you shouldn't import them all as quests);
  • generally, don't import from sources with low signal/noise ratio;
  • don't import stuff just to dump it on questhub if you never visit the website yourself;
  • don't like all quests about chosen topic automatically;

In the future, as questhub grows and develops better ways to filter through large volumes of quests, these rules will be relaxed.

I leave the right to temporarily or permanently revoke the right of anyone's API usage as I see fit.
